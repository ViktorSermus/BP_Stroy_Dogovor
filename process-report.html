Подразделения

  • РП — сквозной контроль; ключевые согласования с ПДО.
  • ПДО — сроки и графики для ОС, Производства, МП.
  • ОС — закупка/логистика только по заданиям ПДО.
  • ОП — проектно-сметная документация, геодезия объекта (ранее в текстах мог встречаться код ПО — в справочнике закреплено ОП).

Регистрация договора Registration

  • Цель: договор в реестрах/ДО (ПТО: Excel, ELMA) и в учёте (Финансы: карточка в 1С7).
  • Инициатор: РП передаёт сведения в ПТО и Финансы.
  • Ход: после РП — ветви ПТО и Финансы моделируются параллельно до полного отражения в системах.
  • Выход: записи ПТО + карточка в 1С7; при необходимости подтверждение РП.
flowchart LR
  RP[РП] --> PTO[ПТО: Excel, ELMA]
  RP --> FIN[Финансы: 1С7]

Стартовое совещание StartSession

  • Цель: после постановки на учёт — согласовать приоритеты и очередность строительства и проектирования.
  • До совещания: РП готовит черновик очередностей.
  • Участники (черновик): РП; руководитель строительного направления (связь с МП/контуром — уточняется); руководитель ОП; руководитель ПДО (обязательность — уточнить); при необходимости менеджер ПДО, руководитель/менеджер ОС; др. по регламенту.
  • Выход: согласованные очередности; понимание следующих шагов для ОП, ПДО и др.; при необходимости протокол/график.

КМ KM

  • Цель: разработка конструкций металлических в ОП по заданию РП сроками; готовность РП получает через руководителя ОП.
  • Цепочка: РП → ОП (в т.ч. КОП по постановке руководителя ОП) → руководитель ОП информирует РП.
  • Открыто: формат задания; промежуточная отчётность; передача КМ в ПДО.
flowchart LR
  A[РП: задание] --> B[ОП / КОП: КМ]
  B --> C[Руководитель ОП]
  C --> D[РП: готовность]

ОЗМ в ELMA OZM

  • Предусловие: без готовности по процессу KM старт ОЗМ не допускается (проверка в ELMA — по настройкам).
  • Зоны: РП (старт, СК при необходимости, проверки КМД/ОЗМ, запуск снабжения, приёмка 1С, стройкомплекты); руководитель ОП (в ELMA дорожка «РОП») и КОП; параллельно учёт: 1С7 (ветка ПДО — товарный учёт или ветка ОПРК) и 1С8 (менеджер производства ОТК); оповещения т.ч. руководитель ПДО.
  • Суть потока: РП → руководитель ОП (подготовка, назначение КОП) → КОП создаёт ОЗМ → проверки, при необходимости график-заказ КМД → заведение в 1С → финальные задачи РП.
  • Уточнить: связь «запустить снабжение» с заданиями ПДО→ОС; правила веток ELMA.
flowchart TB
  R[РП] --> RO[Руководитель ОП]
  RO --> K[КОП: ОЗМ]
  K --> RO
  RO --> R
  R --> Y[Заведение: 1С7 / 1С8]
  Y --> R

РОП в ELMA = руководитель ОП (ОП), отдельного кода «РОП» в справочнике нет.

Геодезия в ELMA Geodez

  • Цель: геодезия по заданию РП; исполнитель в ОП; проверка руководителем ОП; финал РП; статусы и лента ELMA.
  • Поток: старт РП → назначение исполнителя (руководитель ОП) → работы геодезиста → проверка руководителем ОП (возвраты) → проверка РП (возврат руководителю ОП или завершение) → при дедлайнах — ветка «Дедлайн» у руководителя ОП.
  • Выход: файл отчёта по согласованию исполнитель → руководитель ОП → РП; при завершении — уведомления внешним получателям (РК, директор СД — не в departments.md).
flowchart LR
  RP[РП] --> L[Руководитель ОП]
  L --> G[Геодезист]
  G --> L
  L --> RP

Обзор (логика документов)

  • Учёт/регистрация → стартовое совещание → проектные и диспетчерские процессы (КМ, геодезия, ОЗМ и др.).
  • ОЗМ опирается на готовность KM.
flowchart LR
  REG[Регистрация] --> ST[Стартовое совещание]
  ST --> KM[КМ]
  KM --> OZM[ОЗМ ELMA]
  ST --> GZ[Геодезия ELMA]
