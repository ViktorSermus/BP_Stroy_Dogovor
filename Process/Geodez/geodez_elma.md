# Бизнес-процесс: Геодезия (модель в ELMA)

| Поле | Значение |
|------|----------|
| **Идентификатор** | `Geodez` |
| **Название** | Геодезия (процесс в **ELMA**) |
| **Версия** | 0.1 (черновик) |

## 1. Назначение

Обеспечить **выполнение геодезических работ** по объекту/договору по **заданию руководителя проекта (РП)** с **назначением исполнителя** в **отделе проектирования (ОП)**, **проверкой результата** на стороне **руководителя ОП** и **финальным согласованием РП**, с **фиксацией статусов и уведомлениями** в ленте **ELMA**.

В интерфейсе **ELMA** обозначения **«ОП»**, **руководитель ОП**, **сотрудник ОП** соответствуют коду **ОП** и должностям из [departments.md](../../departments.md).

## 2. Справочник подразделений

Роли и коды подразделений по договору заданы в едином файле:

**[departments.md](../../departments.md)**

В этом процессе участвуют **РП**, **ОП** (исполнитель-геодезист и **руководитель ОП** — см. справочник: работы по геодезии объекта отнесены к **ОП**).

## 3. Системы и носители

| Роль / зона | Системы и носители (модель ELMA) |
|-------------|----------------------------------|
| **РП** | Старт процесса, задание на геодезию, финальная проверка и завершение — в **ELMA** (задачи, статусы, лента). |
| **Руководитель ОП** | Назначение исполнителя, проверка материалов геодезии, ветки возврата и дедлайн — в **ELMA**. |
| **Исполнитель (геодезист)** | Выполнение работ и формирование **файла отчёта**, передача на проверку — в **ELMA**; файлы отчёта — по регламенту **ОП** (форматы, хранение — уточнить в регламенте). |

Переходы сопровождаются **сообщениями в ленту** (**ELMA**), чтобы участники видели смену статусов и действия.

## 4. Входы

- **Задание РП** на проведение геодезии (состав требований, объект, сроки — по регламенту компании и полям процесса в **ELMA**).
- Доступные для назначения **исполнители** со стороны **руководителя ОП** (геодезисты или иные роли **ОП** по внутренней модели).

## 5. Основной ход процесса (как в ELMA)

1. **Старт (зона РП).** Служебная задача (присваивание); статус переводится к **руководителю ОП**.
2. **Назначение исполнителя (руководитель ОП).** Задача «Назначить исполнителя»; по результату — сообщение в ленту, статус «у сотрудника ОП» / у исполнителя в **ОП**.
3. **Исполнение (геодезист).** Задача «Создание геодезии»: выполнение работ по заданию **РП**, подготовка **файла отчёта**; отправка **на проверку руководителю ОП** + уведомление в ленту.
4. **Проверка руководителем ОП.** Задача «Проверить геодезию»:
   - при замечаниях — **возврат исполнителю** на доработку (цикл к п. 3), сообщение в ленту, статус обратно исполнителю;
   - при удовлетворительном результате — **передача РП** на проверку, сообщение в ленту.
5. **Проверка РП.** Задача «Проверить геодезию» (зона **РП**):
   - при замечаниях — **возврат руководителю ОП** (цикл к п. 4), сообщение в ленту, статус к **руководителю ОП**;
   - при согласовании с заказчиком (ветка завершения по схеме) — завершение процесса: статус согласования **РП**, затем **автоматическое оповещение** внешних получателей (см. п. 7).

## 6. Дедлайны и просрочка

На задачах **назначения исполнителя**, **создания геодезии** и обеих **проверок** в **ELMA** заданы **контрольные сроки** (таймеры).

При **срыве срока** процесс переходит в ветку статуса **«Дедлайн»** (зона **руководителя ОП**): задача «Проверить по дедлайну» — дальнейшие действия по схеме **ELMA** (возврат к исполнителю или передача на проверку **РП** в зависимости от выбранного перехода).

Детализация правил таймеров и точных переходов — по настройкам процесса в **ELMA**.

## 7. Выходы и уведомления по завершении

- **Файл отчёта** по геодезии (и сопутствующие материалы — по регламенту **ОП**), согласованные по цепочке **исполнитель → руководитель ОП → РП**.
- По успешном завершении: статус **«Согласован РП»** (или эквивалент в **ELMA**) и **автоматическое уведомление** получателей **РК** и **директора СД** (в черновике описания — **внешние получатели уведомления**, без расшифровки ролей в [departments.md](../../departments.md)).

## 8. Ответственные (верхний уровень)

| Роль | Ответственность |
|------|-----------------|
| **РП** | Постановка задачи на геодезию, финальная приёмка, согласование с заказчиком в рамках процесса, завершение экземпляра процесса в **ELMA**. |
| **Руководитель ОП** | Назначение исполнителя, проверка результата геодезии, возвраты на доработку, работа с веткой дедлайна. |
| **Исполнитель (геодезист)** | Выполнение геодезии по заданию **РП**, подготовка отчёта, устранение замечаний по возвратам. |

## 9. Подпроцессы и открытые вопросы

- Внутренние регламенты **ОП** (формат отчёта, именование файлов, связь с проектной документацией) — при необходимости отдельные файлы в `Process/Geodez/`.
- Уточнить при необходимости полный перечень **статусов** и **наименований задач** в **ELMA** (при изменении конфигурации процесса).
- При появлении в организации формализованных кодов для **РК** и **директора СД** — внести в [departments.md](../../departments.md) или вынести в отдельный справочник ролей уведомлений.

## Комментарий ассистента

- **РК** и **директор СД** в черновике не привязаны к строкам [departments.md](../../departments.md); описаны только как получатели финального оповещения.
